# US-001: 手机号注册/登录

## 背景
作为新用户，我希望通过手机号快速注册和登录，以便立即开始使用 App。

## 流程图
```
[输入手机号] → [图形验证码] → [发送短信验证码] → [输入验证码] → [注册/登录成功]
```

## 验收标准

### 注册流程
- [ ] AC1: 输入 11 位手机号，验证格式正确性
- [ ] AC2: 先完成图形验证码，才能发送短信
- [ ] AC3: 短信验证码 6 位数字，有效期 5 分钟
- [ ] AC4: 验证码错误 3 次后需要重新获取
- [ ] AC5: 新用户自动创建账号，老用户直接登录
- [ ] AC6: 登录成功后返回 JWT + Refresh Token

### 界面要求
- [ ] AC7: 手机号输入框自动格式化 (138 0000 0000)
- [ ] AC8: 验证码输入框自动聚焦下一个输入框
- [ ] AC9: 倒计时 60 秒后可重新获取验证码
- [ ] AC10: 错误提示清晰明确 (如：手机号已注册、验证码错误)

### 安全要求
- [ ] AC11: 同一手机号 1 小时内最多发送 5 条短信
- [ ] AC12: 同一 IP 1 小时内最多发送 20 条短信
- [ ] AC13: Token 有效期: Access 2小时, Refresh 7天

## API 依赖
- `POST /api/v1/auth/phone/send-code`
- `POST /api/v1/auth/phone/login`
- `POST /api/v1/auth/token/refresh`
