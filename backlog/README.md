# 需求 Backlog (记忆宫殿的"会议室")

> 所有功能需求按 Epic > Story > Task 层级组织

## 组织规则

```
backlog/
├── README.md (本文件)
├── epics/    # 大功能模块 (如：用户系统、匹配系统)
└── stories/  # 具体用户故事
```

## 优先级标记
- 🔴 **P0** - 阻塞发布，必须完成
- 🟡 **P1** - 重要功能，影响体验
- 🟢 **P2** - 优化项，可后续迭代

## 新建 Story 模板

创建新需求时，复制以下模板：

```markdown
# US-XXX: 需求标题

## 背景
作为 [角色]，我希望 [功能]，以便 [价值]

## 验收标准 (AC)
- [ ] AC1: 具体可验证的条件
- [ ] AC2: 具体可验证的条件
- [ ] AC3: 具体可验证的条件

## 技术 Notes
- 相关接口:
- 依赖 Story:
- UI 设计稿:

## 估计工作量
- 开发: X 天
- 测试: X 天

## 状态历史
- 2026-04-09: 创建
```
