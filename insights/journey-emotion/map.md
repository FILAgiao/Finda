# Finda 用户旅程情绪地图

> 基于峰终定律和情绪设计理论，优化关键触点

## 需求方旅程 (Seeker Journey)

```
情绪强度
  +3 │                                    ★结束
     │                                   ／ 满足
  +2 │                              匹配成功
     │                             ／
  +1 │         首次打开              ／
     │           好奇        发布完成  ／
   0 ├────────────────────────────────────────────────────
     │                    等待响应    选择困难
  -1 │                   焦虑│││    ／
     │                  ／          ／
  -2 │    注册繁琐─────
     │   挫败
  -3 │
     └────────────────────────────────────────────────────
       发现   注册   发布   等待   选择   沟通   完成   复购
       App   登录   需求   响应   匹配   确认   服务   留存
```

### 阶段详解与优化策略

#### 1. 发现 App (情绪: 好奇)
**用户心理**: 这是什么？能解决我的问题吗？

**关键触点**:
- 应用商店页面
- 首次打开引导

**心理学优化**:
```markdown
现状: "智能匹配服务平台"
优化: "3分钟找到靠谱师傅，不满意全额退"

原理: 
- 具体数字（3分钟）> 抽象概念（智能）
- 损失保障（全额退）> 功能描述
- 目标人群（师傅）> 泛化表述
```

**产品需求**:
- [ ] US-J101: 应用商店页 A/B 测试（功能导向 vs 结果导向）
- [ ] US-J102: 首屏直接展示"解决什么问题"而非功能列表

---

#### 2. 注册登录 (情绪: 轻微挫败 → 需要平滑)
**用户心理**: 又要填一堆信息？能不能快点？

**心理学优化**:
```markdown
现状: 手机号 → 验证码 → 密码 → 个人信息
优化: 手机号 → 验证码 → 立即使用（信息后续补全）

原理:
- 延迟满足: 先给甜头，再要信息
- 渐进承诺: 每完成一步增加沉没成本
```

**产品需求**:
- [ ] US-J201: 延迟注册 - 先浏览/发布，后补全信息
- [ ] US-J202: 一键本机号码登录（减少输入）
- [ ] US-J203: 注册进度可视化（已完成 1/3）

---

#### 3. 发布需求 (情绪: 期待 → 可能困惑)
**用户心理**: 我能说清楚我要什么吗？会被理解吗？

**心理学优化**:
```markdown
现状: 自由文本输入
优化: 结构化引导 + 智能补全

示例交互:
"您需要什么服务？"
→ 选择: 维修 / 家政 / 设计 / ...

"具体是？"
→ 选择: 水管 / 电路 / 门窗 / ...

"紧急程度？"
→ 滑动条: 今天 ←→ 本周

原理:
- 选项有限化（减少决策负担）
- 渐进披露（一次只问一个问题）
- 即时反馈（每步都显示进度）
```

**产品需求**:
- [ ] US-J301: 需求发布向导（Wizard 模式）
- [ ] US-J302: 拍照识别需求（AI 解析）
- [ ] US-J303: 相似需求案例参考（社会认同）

---

#### 4. 等待响应 (情绪: 焦虑 ⚠️ 关键负面点)
**用户心理**: 有人看到吗？我要等多久？是不是该换平台？

**心理学优化**:
```markdown
现状: "发布成功，等待响应"
优化: "已推送给附近8位师傅，平均3分钟内响应"

进阶:
- 实时状态: "张师傅正在查看..."
- 社会证明: "李阿姨2分钟前完成了类似需求"
- 控制感: "如需加急，可提升赏金"

原理:
- 不确定性消除: 知道发生什么减少焦虑
- 进度可视化: 看到进展增强耐心
- 逃生通道: 感觉有控制就不那么焦虑
```

**产品需求**:
- [ ] US-J401: 实时推送状态可视化
- [ ] US-J402: 等待时展示"成功案例"
- [ ] US-J403: 预计响应时间算法 + 显示
- [ ] US-J404: 超时时自动升级匹配范围

---

#### 5. 选择匹配 (情绪: 选择困难 ⚠️ 关键负面点)
**用户心理**: 这么多怎么选？选错了怎么办？

**心理学优化**:
```markdown
现状: 列表展示5-10个服务商
优化: 
1. AI推荐"最适合"（仅1个）
2. "其他备选"折叠在下方
3. 推荐理由:"距离最近、价格适中、评价优秀"

原理:
- 默认选项: 减少选择负担
- 理由透明: 增加信任
- 层次化: 主推荐 vs 备选
```

**产品需求**:
- [ ] US-J501: AI 单推荐模式（默认只展示1个）
- [ ] US-J502: 推荐理由可视化
- [ ] US-J503: "帮我选择"功能（根据历史偏好自动选）

---

#### 6. 沟通确认 (情绪: 谨慎 → 建立信任)
**用户心理**: TA靠谱吗？会不会中途加价？

**心理学优化**:
```markdown
- 展示"平台担保"徽章
- 预设常见问题快捷回复
- 价格确认表单（避免口头纠纷）
- 历史成交数据展示

原理:
- 权威背书: 平台担保降低风险感知
- 结构化沟通: 减少误解
- 数据透明: 历史记录建立信任
```

**产品需求**:
- [ ] US-J601: 结构化沟通模板
- [ ] US-J602: 报价确认书（双方电子确认）
- [ ] US-J603: 风险提醒（常见诈骗模式警示）

---

#### 7. 服务完成 (情绪: 满意 ★ 峰值点)
**用户心理**: 终于完成了！体验如何？

**心理学优化**:
```markdown
- 完成庆祝动效（ confetti effect ）
- 即时评价引导（趁热打铁）
- 感谢服务商的话术模板
- 分享成就:"又解决了一个问题！"

原理:
- 峰终定律: 结束时的愉悦感强化整体记忆
- 即时反馈: 行为后立即有结果
- 社交货币: 让用户有分享的素材
```

**产品需求**:
- [ ] US-J701: 服务完成庆祝动效
- [ ] US-J702: 一键评价（5星快速评价）
- [ ] US-J703: 精美评价卡片生成（可分享）

---

#### 8. 复购留存 (情绪: 熟悉 → 忠诚)
**用户心理**: 下次还找TA / 继续用这个平台

**心理学优化**:
```markdown
- "欢迎回来，张师傅本月还有空档"
- 会员等级体系（成长感）
- 专属优惠（老用户特权）

原理:
- 熟悉感: 降低再次决策成本
- 沉没成本: 历史订单形成迁移成本
- 专属感: 被区别对待的感觉
```

**产品需求**:
- [ ] US-J801: 首页展示"常用服务商"
- [ ] US-J802: 会员成长体系
- [ ] US-J803: 老用户专属需求通道

---

## 服务方旅程 (Provider Journey)

### 核心差异
- **动机更复杂**: 不只是赚钱，还有时间灵活、职业尊严
- **恐惧更深层**: 怕平台剥削、怕客户刁难、怕收入不稳定
- **关键情绪点**: 第一单成交（验证自我价值）

### 关键优化点

| 阶段 | 心理 | 优化策略 |
|------|------|---------|
| 注册成为服务商 | "我能行吗？" | 降低门槛，强调"技能即资本" |
| 等待第一单 | "真的有单吗？" | 新手保护期，优先派单 |
| 报价竞争 | "价格压太低了" | 价值导向而非价格导向 |
| 服务中 | "客户难搞怎么办" | 平台介入机制明确 |
| 收到好评 | "被认可了！" | 即时反馈 + 成就系统 |
| 收入提现 | "钱到账了吗" | 实时到账 + 收入可视化 |
