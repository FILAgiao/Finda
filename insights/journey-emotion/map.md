# Finda 用户旅程情绪地图

> 基于峰终定律和情绪设计理论，优化关键触点
> **产品**: AI约会/饭搭撮合平台 —— "一天只推一个人，月老点名就是他"

## 主动方旅程 (Initiator Journey)

```
情绪强度
  +3 │                                    ★结束
     │                                   ／ 满足
  +2 │                              匹配成功
     │                             ／
  +1 │         首次打开              ／
     │           好奇        发布需求  ／
   0 ├────────────────────────────────────────────────────
     │                    等待匹配    被拒失落
  -1 │                   焦虑│││    ／
     │                  ／          ／
  -2 │    注册繁琐─────
     │   挫败
  -3 │
     └────────────────────────────────────────────────────
       发现   注册   发布   等待   收到   见面   完成   复购
       App   登录   需求   匹配   确认   约会   评价   留存
```

### 阶段详解与优化策略

#### 1. 发现 App (情绪: 好奇)
**用户心理**: 这是什么？能真的帮我找到饭搭子吗？

**关键触点**:
- 应用商店页面
- 首次打开引导

**心理学优化**:
```markdown
现状: "AI配对，你想要搭子，月老都懂你"
优化: "一天只推一个人，今晚就有人陪你吃饭"

原理: 
- 具体结果（今晚吃饭）> 抽象概念（AI配对）
- 确定性（一定有人）> 可能性（可能找到）
- 稀缺性（一天一个）创造独特性
```

**产品需求**:
- [ ] US-J101: 应用商店页 A/B 测试（抽象概念 vs 具体结果）
- [ ] US-J102: 首屏展示"今晚有人陪你吃饭"

---

#### 2. 注册登录 (情绪: 轻微挫败 → 需要平滑)
**用户心理**: 又要填一堆信息？能不能快点？

**心理学优化**:
```markdown
现状: 手机号 → 验证码 → 填资料 → 上传照片
优化: 手机号 → 验证码 → 极简资料 → 立即使用

核心:
- 延迟完善: 先给甜头，再要信息
- 身份认证后置: 先体验，再验证（但认证后有标识）
```

**产品需求**:
- [ ] US-J201: 延迟注册 - 先浏览/发布，后完善资料
- [ ] US-J202: 一键本机号码登录
- [ ] US-J203: 公务员/教师身份认证标识（增加信任）

---

#### 3. 发布需求 (情绪: 期待)
**用户心理**: 我想今晚吃火锅，能找到人吗？

**心理学优化**:
```markdown
极简输入:
"你想吃什么？"
→ 选择: 火锅 / 日料 / 西餐 / ...

"什么时候？"
→ 选择: 今晚 / 明天 / 本周末

"预算？"
→ 滑动条: AA ←→ 我请客

AI自动补全:
- 基于历史偏好推荐餐厅
- 基于位置推荐就近
- 基于时间匹配合适对象

原理:
- 选项有限化（减少决策负担）
- AI辅助（不用写小作文）
- 即时反馈（显示"正在为你匹配"）
```

**产品需求**:
- [ ] US-J301: 极简需求发布（3步完成）
- [ ] US-J302: AI智能推荐餐厅
- [ ] US-J303: 诚意金支付（表达真实意图）

---

#### 4. 等待匹配 (情绪: 焦虑 ⚠️ 关键负面点)
**用户心理**: AI月老在找吗？能找到人吗？是不是该换软件？

**心理学优化**:
```markdown
现状: "发布成功，等待匹配"
优化: "AI月老正在全城为你物色最佳人选..."

进阶:
- 进度可视化: "已筛选50位 → 已匹配3位 → 正在计算最佳"
- 预期管理: "平均匹配时间5分钟"
- 控制感: "如需加急，可提升诚意金"

原理:
- 不确定性消除: 知道发生什么减少焦虑
- 进度可视化: 看到进展增强耐心
- AI人格化: "月老"比"系统"更有温度
```

**产品需求**:
- [ ] US-J401: AI月老匹配进度可视化（🔴 P0）
- [ ] US-J402: 预计匹配时间算法 + 显示
- [ ] US-J403: 匹配失败安抚（"缘分未到，明天再来"）

---

#### 5. 收到确认 (情绪: 兴奋 ★ 关键正面点)
**用户心理**: TA接受了！真的有人愿意和我吃饭！

**心理学优化**:
```markdown
- 庆祝动效: "月老撮合成功！"
- 专属感: "你是TA今天的唯一选择"
- 破冰辅助: AI提供今日话题
- 实用信息: 时间、地点、联系方式

原理:
- 峰值体验: 匹配成功是最大爽点
- 社交货币: 可分享的成就卡片
- 降低尴尬: AI辅助破冰
```

**产品需求**:
- [ ] US-J501: 匹配成功庆祝动效（🟡 P1）
- [ ] US-J502: "月老撮合"文案与动效
- [ ] US-J503: AI话题推荐（破冰辅助）

---

#### 6. 见面约会 (情绪: 紧张 → 愉悦)
**用户心理**: 照片真实吗？聊什么不尴尬？

**心理学优化**:
- 颜值检测已验证照片真实性
- AI话题推荐打破冷场
- 诚意金机制确保双方认真赴约
- 地理围栏确认到达

**产品需求**:
- [ ] US-J601: AI话题推荐
- [ ] US-J602: 到达确认（地理围栏）
- [ ] US-J603: 紧急求助按钮（安全保障）

---

#### 7. 完成评价 (情绪: 满意)
**用户心理**: 体验如何？要不要继续联系？

**心理学优化**:
```markdown
- 简化评价: 3星快速评价
- 私密性: 评价仅系统可见，不对对方公开
- 反馈闭环: "感谢你的反馈，会帮你匹配更合适的人"
- 二次约会: "想再见TA？点击表达意向"

原理:
- 峰终定律: 结束时的愉悦感
- 隐私保护: 鼓励真实评价
- 持续优化: 评价数据改进匹配算法
```

**产品需求**:
- [ ] US-J701: 极简评价（3星制）
- [ ] US-J702: 私密评价机制
- [ ] US-J703: 二次约会意向表达

---

#### 8. 复购留存 (情绪: 期待)
**用户心理**: 下次什么时候有匹配？

**心理学优化**:
- "明天同一时间，月老继续为你撮合"
- 会员等级: 等级越高匹配质量越高
- 历史回顾: "本月已认识5位新朋友"

**产品需求**:
- [ ] US-J801: 明日匹配预告
- [ ] US-J802: 会员成长体系
- [ ] US-J803: 社交成就统计

## 响应方旅程 (Responder Journey)

### 核心差异
- **被动等待**: 不是主动发布，而是等待被邀请
- **惊喜感**: "有人想请你吃饭"是惊喜
- **选择权**: 有明确的接受/拒绝权利

### 关键优化点

| 阶段 | 心理 | 优化策略 |
|------|------|---------|
| **收到推送** | "有人想请我？" | 精美推送，专属感文案 |
| **查看资料** | TA靠谱吗？ | 诚意金标识、身份认证 |
| **决策** | 要不要去？ | 明确接受/拒绝按钮，无负担 |
| **接受后** | 紧张期待 | 破冰辅助、话题推荐 |
| **见面后** | 评价 | 私密评价、二次约会选项 |
