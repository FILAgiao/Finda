# Finda 增长实验设计

> 基于心理学假设的数据驱动实验

## 实验方法论

### 福格模型 + A/B 测试
```
每个实验验证一个心理假设:
假设 → 设计实验 → 数据验证 → 规模化 / 放弃
```

### 实验状态流转
```
💡 想法 → 📋 设计中 → 🔬 测试中 → 📊 分析中 → ✅ 规模化 / ❌ 放弃
```

## 当前进行中的实验

### EXP-001: 等待状态可视化是否能减少焦虑
**假设**: 显示"已推送给8位师傅"比"等待响应"能减少30%的取消率

**实验设计**:
```yaml
目标指标: 发布需求后30分钟内取消率
对照组:   "发布成功，等待响应"
实验组:   "已推送给附近8位师傅，平均3分钟响应"
实验时长: 2周
样本量:   1000次发布/组
成功标准: 取消率降低 > 15%（达到统计显著性）
```

**心理学原理**:
- 不确定性消除理论
- 进度可视化增强耐心

**关联产品需求**: US-J401

**状态**: 📋 设计中

---

### EXP-002: AI 单推荐 vs 列表选择
**假设**: 只推荐1个"最佳匹配"比展示列表能提升20%的匹配成功率

**实验设计**:
```yaml
目标指标: 查看推荐后的确认预约率
对照组:   列表展示5个服务商
实验组:   AI单推荐+备选折叠
实验时长: 3周
样本量:   500次匹配/组
成功标准: 确认率提升 > 10%
```

**心理学原理**:
- 决策瘫痪 (Choice Paralysis)
- 默认选项效应

**关联产品需求**: US-J501

**状态**: 💡 想法

---

### EXP-003: 完成庆祝动效对评价率的影响
**假设**: 服务完成后播放庆祝动效能提升15%的评价率

**实验设计**:
```yaml
目标指标: 服务完成后24小时内评价率
对照组:   静态"服务完成"提示
实验组:   庆祝动效+正向反馈文案
实验时长: 2周
样本量:   800次完成/组
成功标准: 评价率提升 > 10%
```

**心理学原理**:
- 峰终定律 (Peak-End Rule)
- 即时正向反馈

**关联产品需求**: US-J701

**状态**: 💡 想法

### EXP-004: 损失框架 vs 收益框架
**假设**: 强调"避免踩坑"比"找到好服务"能提升注册转化率

**实验设计**:
```yaml
目标指标: 应用商店到注册完成率
对照组:   "3分钟找到好师傅"
实验组:   "避免90%的人会踩的坑"
实验时长: 4周
样本量:   2000次点击/组
成功标准: 注册率提升 > 8%
```

**心理学原理**:
- 损失厌恶 (Loss Aversion)
- 框架效应 (Framing Effect)

**关联产品需求**: US-J101

**状态**: 💡 想法
