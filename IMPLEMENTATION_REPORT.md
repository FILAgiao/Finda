## 2. 项目背景

### 2.1 Finda 产品定位
**Finda 是一款 AI 驱动的约会/饭搭撮合平台**，解决传统约会软件的低效问题：

**传统约会软件的问题**:
- 左滑右滑 → 选择困难，永远觉得"下一个更好"
- 聊天预热 → 聊一周不见面，浪费时间
- 被鸽率高 → 对方不认真，临时爽约
- 隐私暴露 → 大广场晒照片，公务员/教师不敢用

**Finda 的解决方案**:
- **一天只推一个人** → 没得选，只能认真考虑
- **即时撮合** → 基于"今晚想吃火锅"实时匹配
- **诚意金质押** → 爽约方赔付，确保认真
- **暗池模式** → 信息仅匹配双方可见

### 2.2 原有状况
- 已有 Android/iOS 原生代码库
- 已有基础技术文档（架构、算法、竞品分析）
- 缺少系统化的需求管理和用户洞察
- **需要明确**: 产品定位是约会撮合，不是服务撮合

### 2.3 目标
建立一套完整的项目管理体系：
1. 结构化需求管理（Backlog）
2. 技术决策记录（ADR）
3. 约会场景用户心理洞察（Psychology Map）
4. 增长实验设计（Experiments）
5. AI 协作规则（.cursorrules）

### 3.2 心理学洞察体系（核心创新）

#### 3.2.1 双边用户心理画像（已修正）

**主动方 (Initiator)** - 想主动发起约会的人:

| 心理机制 | 理论来源 | 产品启示 |
|---------|---------|---------|
| 损失厌恶 | Kahneman & Tversky | 强调"防鸽""诚意金保障" |
| 决策瘫痪 | Iyengar & Lepper | **每天只推1个人** |
| 社会身份焦虑 | Social Identity | **暗池模式**保护隐私 |
| 反荡妇机制 | Self-Presentation | AI中介，需求隐式匹配 |
| 即时满足 | Mischel | 即时撮合，今晚见面 |

**响应方 (Responder)** - 等待被邀请的人:

| 心理机制 | 表现 | 产品启示 |
|---------|------|---------|
| 稀缺性与专属感 | "我只被推荐给1个人" | "你是TA今天的唯一选择" |
| 安全感 | 担心骗子/照骗 | 诚意金+身份认证 |
| 控制感 | 希望有拒绝权 | 明确接受/拒绝按钮 |

#### 3.2.2 用户旅程情绪地图（约会场景）

**主动方关键情绪点**:
1. **发布需求** → 期待
2. **等待匹配** → 焦虑 ⚠️ (需优化)
3. **收到确认** → 兴奋 ★ (峰值点)
4. **见面约会** → 紧张→愉悦
5. **完成评价** → 满意

**响应方关键情绪点**:
1. **收到推送** → 惊喜 ★ ("有人想请我吃饭")
2. **查看资料** → 评估
3. **决策** → 选择困难 ⚠️ (需简化)
4. **接受后** → 期待
5. **见面后** → 评价

**关键优化策略**:
- 等待焦虑 → AI月老进度可视化
- 选择困难 → 每天只推1个人
- 被拒失落 → "缘分未到，明天再来"

## 4. 关键创新点

### 4.1 约会撮合 vs 服务撮合的本质区别

| 维度 | 服务撮合 | 约会撮合 |
|------|---------|---------|
| 交易性质 | 单向（我买你服务） | 双向（互相选择） |
| 核心痛点 | 质量不确定 | 被鸽、选择困难、隐私暴露 |
| 解决方案 | 评价系统 | 诚意金、每天1人、暗池模式 |
| 成功标准 | 服务完成 | 双方满意、可能二次约会 |

### 4.2 核心差异化功能

1. **每天只推一个人**
   - 解决选择困难
   - 创造稀缺性和专属感
   - 迫使用户认真考虑

2. **诚意金质押**
   - 解决"被鸽"问题
   - 爽约方赔付，筛选认真用户
   - 建立信任基础

3. **暗池模式**
   - 保护隐私，区别于Tinder
   - 吸引公务员/教师等精英用户
   - 降低社交压力

4. **AI月老撮合**
   - 基于意图匹配，而非外貌滑动
   - "今晚想吃火锅"→匹配合适对象
   - 降低反荡妇/反骚男机制

## 6. 使用指南

### 6.1 日常开发流程

```bash
# 1. 查看项目状态
./finda.sh status

# 2. 了解全局上下文
cat PLAN.md

# 3. 理解用户心理（约会场景）
cat insights/psychology-map/persona.md

# 4. 查看要做的 Story
./finda.sh story US-XXX

# 5. 创建开发任务
./finda.sh task create TASK-XXX

# 6. 编码实现（关联 Task ID）

# 7. 标记任务完成
./finda.sh task done TASK-XXX
```

### 6.2 规划新功能流程

```
1. 识别约会场景痛点
   → 查看 insights/journey-emotion/map.md
   → 找到情绪低谷点（如等待焦虑）

2. 设计实验验证假设
   → 创建 insights/experiments/EXP-XXX.md

3. 实验成功 → 创建 Story
   → backlog/stories/US-XXX.md

4. 拆解为 Task
   → .claude/tasks/TASK-XXX.md

5. 开发实现

6. 收集数据验证假设
```

## 7. 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 诚意金用户不接受 | 高 | A/B测试不同金额，从低到高 |
| 每日1人用户觉得太少 | 中 | 提供"加急"选项，额外付费 |
| 暗池模式冷启动困难 | 高 | 早期引入KOL/种子用户 |
| 被误认为约炮软件 | 高 | 强调"饭搭"概念，阳光形象 |
